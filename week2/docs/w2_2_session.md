## 이번 세션 핵심 개념

### 1. Transform의 3단계: 원본 → 중간 → 비즈니스

데이터는 보통 한 번에 최종 결과물이 되지 않고,  
목적에 따라 여러 단계를 거쳐 변환됩니다.

#### 1) 원본 테이블(raw)
- 아직 손대지 않은 원본 데이터
- 결측치, 자료형 문제, 중복, 이상치가 남아 있을 수 있음
- 바로 분석하거나 시스템에 넣기엔 불안할 수 있음

**영화 데이터 예시**
- `movies_metadata.csv`
- `keywords.csv`

#### 2) 중간 테이블(cleaned / refined)
- 원본 데이터를 바로 쓰지 않고, 한 번 정리한 테이블
- 분석이나 적재가 가능하도록 기본 품질을 맞춘 상태
- 다음 단계에서 다시 활용하기 좋은 형태

**영화 데이터 예시**
- `movies_clean`
- `movies_with_keyword`

#### 3) 비즈니스 테이블(business-ready)
- 사람이 바로 읽거나, BI/시각화에서 바로 쓸 수 있도록 만든 최종 결과물
- 이미 목적이 어느 정도 정해져 있고, 그 목적에 맞는 지표와 구조를 가짐

**영화 데이터 예시**
- 연도별 고평점 영화 수 요약표
- 대표 키워드별 영화 수 / 평균 평점 / 평균 수익 테이블
- 연대 × 대표 키워드 pivot table

---

### 2. 중간 테이블에서 하는 일

중간 테이블에서는 크게 두 종류의 작업이 일어납니다.

#### 1) 품질 정비
- 결측치 처리
- 중복 제거
- 자료형 맞추기
- 표준화
- 기본 검증

즉,  
**“이 데이터를 믿고 써도 되나?”** 를 확인하는 단계입니다.

#### 2) 목적에 맞는 변환
- 컬럼 선별 / 삭제
- 조인(join)
- 파생 컬럼 생성
- 집계
- reshaping

즉,  
**“이 데이터를 다음 단계가 쓰기 좋게 어떻게 바꿀까?”** 를 고민하는 단계입니다.

---

### 3. pandas에서의 활용

#### `groupby + agg`
- 같은 기준끼리 묶고(`groupby`)
- 여러 통계값을 계산하는(`agg`) 방식
- 추가 계산이나 재가공에 유리한 요약표를 만들기 좋음

#### `pivot_table()`
- 사람이 비교해서 읽기 좋은 표를 만들기 좋음
- 행과 열을 교차시켜 한눈에 패턴을 보기 좋게 만듦

#### `melt()`
- wide 형태의 표를 다시 long 형태로 바꾸는 함수
- 추가 가공, 시각화, 후속 처리에 더 유리한 구조를 만들 때 사용

---

### 4. wide vs long

같은 데이터라도 목적에 따라 더 잘 맞는 형태가 달라질 수 있습니다.

#### wide
- 사람이 표를 보고 비교하기 좋은 형태
- 보고서나 공유용 결과물에 자주 쓰임

#### long
- 시각화, 추가 집계, 후속 처리에 더 유리한 형태
- 여러 지표를 같은 구조로 다루기 쉬움

즉, 데이터의 내용은 같아도  
**누가 볼 것인지, 다음에 무엇을 할 것인지** 에 따라 표의 모양이 달라질 수 있습니다.

## 이번 세션 큰 흐름

이번 시간은 먼저 질문을 정하고, 그 질문에 답하기 위해 테이블을 변환하는 흐름으로 진행됩니다.

### 이번 시간에 풀 질문

1. 신뢰할 만한 고평점 영화는 어느 시기에 많았을까?
2. 대표 키워드별로 수익성과 효율은 어떻게 다를까?
3. 시기별로 어떤 대표 키워드가 많이 등장했을까?
4. wide table과 long table은 각각 언제 쓰기 좋을까?

이 질문들 때문에 `keywords.csv`를 붙이고, `profit`, `roi`, `release_decade` 같은 파생 컬럼을 만듭니다.

### 1. 중간 테이블 보강하기
저번 시간에 만든 `movies_clean`에  
키워드 정보를 붙여서 **더 풍부한 중간 테이블**을 만듭니다.

여기서는 다음 흐름을 봅니다.

- `movies_clean` 불러오기
- `keywords`에서 대표 키워드(`main_keyword`) 추출
- `keywords_clean` 정리
- `movie_id` 기준 조인
- `profit`, `roi`, `release_decade` 파생 컬럼 생성
- `movies_with_keyword` 생성

핵심은  
**join과 파생 컬럼 생성이 중간 테이블을 더 풍부하게 만드는 과정일 수 있다** 는 점입니다.

참고로 이번 데이터에서는 `movies_clean`과 `keywords_clean`이 대부분 같은 `movie_id`를 공유하므로 `inner join`을 사용합니다.  
`main_keyword`가 비어 있는 영화는 join 방식으로 제거하는 것이 아니라, 키워드 분석 단계에서 필요할 때 필터링합니다.

---

### 2. 비즈니스 테이블 만들기
보강된 중간 테이블을 바탕으로  
초반에 정한 비즈니스 질문에 답하는 최종 결과물을 만듭니다.

예를 들어 이런 흐름으로 진행합니다.

- `vote_average`와 `vote_count`를 함께 사용해 신뢰 가능한 고평점 영화 필터링
- 고평점 영화가 많은 연도 요약
- 연도별 영화 수 / 평균 평점 / 평균 투표 수 / 평균 수익 요약
- 연대 × 대표 키워드 pivot table 생성
- 대표 키워드별 영화 수 / 평균 평점 / 평균 수익 / 평균 이익 / 평균 ROI 집계
- wide table을 `melt()`로 long table로 변환

이 과정에서
- `groupby + agg`
- `pivot_table()`
- `melt()`
를 목적에 맞게 사용합니다.
